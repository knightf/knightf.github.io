Generator的next方法

June 1, 2016
JavaScript Generator ES6

使用function*声明的Generator函数，其实并不是立即执行的函数，它返回一个Generator对象。这个对象主要的方法就是next方法。使用next方法就可以一步步的执行这个Generator对象中的语句。

function* gen() { 
  yield 1;
  yield 2;
  yield 3;
}

var g = gen();

上面的代码中g就是一个Generator对象，不断调用它的next方法，就会依次输出1、2和3。调用next方法，返回的是一个对象，这个对象有两个属性：

  • done: 布尔值，表示这个Generator对象是不是已经没有更多代码可以执行了。
  • value: 就是yield关键字后面表达式执行的结果。

同时，next方法还有一个形参value。如果在调用next方法的时候给定了这个形参，那么在这一步的执行过程中，yield后面表达式的值就会被替换成这个形参，所以一次next方法执行的顺序可以归纳成：

  1. 执行yield关键字后面的表达式得到值temp
  2. 如果next方法传入了value，temp = value
  3. 包含yield关键字的这一完整表达式，值为temp

需要注意的一点是，当执行Generator函数得到Generator之后，这个对象中的第一段代码块并没有执行，一定要执行一次next方法才会真正启动这个Generator。
